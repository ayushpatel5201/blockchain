Architecture

How Sharding Works

01
⛓

Network Partitioning

The network is divided into N shards, each responsible for a subset of accounts and transactions. Validators are randomly assigned to shards to prevent collusion.

02
⚡

Intra-Shard Consensus

Each shard runs its own BFT consensus independently in parallel. This multiplies throughput linearly — 4 shards = ~4× the TPS of a single chain.

03
🔗

Cross-Shard Communication

Transactions spanning multiple shards use a secure relay protocol with cryptographic receipts, ensuring atomicity without a centralized coordinator.

04
🛡

Security Preservation

Random validator reassignment each epoch prevents targeted attacks. Each shard maintains full Merkle proof support for cross-chain verification.
